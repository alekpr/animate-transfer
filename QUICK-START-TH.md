## 📝 ขั้นตอนที่ 1: Animation (Core Features)

### 1. เปิด Workflow
```
โหลดไฟล์: lipsync-ofm+Nabludatel-24GB-NoRIFE.json
```

### 2. อัปโหลดวิดีโอ
- คลิก Node: VHS_LoadVideo (Node 75)
- เลือกวิดีโอต้นฉบับ
- แนะนำ: 10-15 วินาทีสำหรับทดสอบ

### 3. กด Queue Prompt
- VRAM ที่ใช้: **~15-18GB** ✅
- เวลา: **~3-5 นาที** (10 วินาที)
- Output: `lipsync_output_00001.mp4`

### 4. ฟีเจอร์ที่ได้
- ✅ Lip Sync (ปากสอดคล้องกับเสียง)
- ✅ Face Animation (ใบหน้าเคลื่อนไหว)
- ✅ Pose Detection (ตรวจจับท่าทาง)
- ✅ Hand Detection (ตรวจจับมือ)
- ❌ Frame Interpolation (ปิดเพื่อประหยัด VRAM)

### 5. ที่เปลี่ยนจากเดิม
**Node 360 (RIFE Interpolation) ถูก BYPASS**
- เหตุผล: ประหยัด VRAM ~3-5GB
- ผลกระทบ: ได้ 24fps ตามต้นฉบับ (ไม่มีการเพิ่ม frame)
- แก้ไข: ไปเพิ่มใน ขั้นตอนที่ 2 แทน

---

## ✨ ขั้นตอนที่ 2: Upscale + Smooth (Optional)

### 1. เปิด Workflow
```
โหลดไฟล์: Video-Upscale-Enhanced.json
```

### 2. อัปโหลด Output จาก ขั้นตอนที่ 1
- คลิก Node: Load Video (Node 1)
- เลือกไฟล์: `lipsync_output_00001.mp4`

### 3. ตั้งค่า Upscale
- Node 5 (Upscale Mode): เลือก **1** (2x upscale)
  - 0 = ไม่ขยาย
  - 1 = ขยาย 2 เท่า ← **แนะนำ**
  - 2 = Fix 1080P

### 4. ตั้งค่า RIFE (เพิ่มความลื่นไหล)
- Node 10 (Enable RIFE): **true** (เปิด)
- Node 12 (RIFE Multiplier): **2** (เพิ่ม FPS 2 เท่า)
- Node 13 (Output FPS): **48** (คำนวณ: 24 × 2 = 48)

⚠️ **สำคัญ!** Output FPS ต้องคำนวณเอง:
```
Output FPS = Source FPS × RIFE Multiplier
ตัวอย่าง: 24 × 2 = 48
```

### 5. กด Queue Prompt
- VRAM ที่ใช้: **~10-13GB** ✅
- เวลา: **~3-4 นาที** (10 วินาที)
- Output: `upscaled_enhanced_00001.mp4`

### 6. ผลลัพธ์ที่ได้
- 📐 Resolution: 1280×2272 (2x)
- 🎬 FPS: 48fps (smooth!)
- 💾 File size: ~90-120MB per minute
- ⭐ Quality: HD + ลื่นไหล

## 🆘 แก้ปัญหา

### ปัญหา: ขั้นตอนที่ 1 ยัง OOM อยู่
**วิธีแก้**:
```
1. ลด frame_load_cap: 120 → 60 (Node 75)
2. ลด context_frames: 49 → 33 (Node 87)
3. ลดความละเอียด: 640×1136 → 576×1024
```

### ปัญหา: ขั้นตอนที่ 2 วิดีโอเร็ว/ช้าเกินไป
**วิธีแก้**:
```
ตรวจสอบ Output FPS (Node 13)
ต้องเป็น: Source FPS × RIFE Multiplier
ตัวอย่าง: 24 × 2 = 48 fps
```

### ปัญหา: เสียงไม่ตรงกับภาพ
**วิธีแก้**:
```
1. ตรวจสอบ Output FPS ว่าถูกต้อง
2. ตรวจสอบว่าวิดีโอต้นฉบับมีเสียง
3. ตรวจสอบ audio link ใน workflow
```
