## 🎯 สิ่งที่ปรับปรุง

### ลดการใช้ VRAM จาก 26-32GB → 19-23GB ✅

**การเปลี่ยนแปลงหลัก 11 จุด**:

1. ✅ **ลดความละเอียด** 720×1280 → 640×1136 (ลด 22% pixels)
2. ✅ **เปิด VAE Tiling** ประมวลผลทีละ tile 256×256 แทนทั้งเฟรม
3. ✅ **ลด Context Frames** จาก 81 → 49 frames
4. ✅ **เปิด Force Offload** ใน WanVideoAnimateEmbeds
5. ✅ **เปิด Low Memory LoRA** โหลดแบบ on-demand
6. ✅ **ลด Frame Buffer** จาก 241 → 120 frames

**ประหยัด VRAM รวม**: 9-13GB  
**ฟีเจอร์ที่รักษาไว้**: ครบทั้งหมด (lip sync, face animation, pose/hand detection, interpolation)

## 🚀 วิธีใช้งาน

### สำหรับวิดีโอ ≤10 วินาที (300 frames)
✅ **ใช้ได้เลย** - ไม่ต้องแบ่ง chunk  
โหลดไฟล์ `lipsync-ofm+Nabludatel-24GB.json` ใน ComfyUI แล้วรันได้เลย

### สำหรับวิดีโอ 11-20 วินาที (300-600 frames)
⚠️ **แนะนำ**: แบ่งประมวลผลเป็น 2 ช่วงๆ ละ 10 วินาที แล้วนำมาต่อกัน

### สำหรับวิดีโอ 21-30 วินาที (600-900 frames)
⚠️ **ต้องแบ่ง chunk** - ประมวลผลเป็นช่วงๆ

**วิธีแบ่ง chunk แนะนำ**:
```
วิดีโอ 30 วินาที = 900 frames

แบ่งเป็น 8 chunks:
- Chunk 1: frames 0-112
- Chunk 2: frames 104-216   (overlap 8 frames)
- Chunk 3: frames 208-320
... และต่อไปเรื่อยๆ
```

**การต่อ chunk**:
1. รันแต่ละ chunk แยกกัน
2. ใช้ linear blending ที่ตำแหน่ง overlap (8 frames)
3. นำ chunks มาต่อกันเป็นวิดีโอเดียว
4. ใส่เสียงกลับเข้าไป

## 🔧 แก้ปัญหาเมื่อ VRAM ยังเกิน 24GB

ลองทำตามลำดับ:

### 1. ลด Context Frames เหลือ 33
- แก้ไข Node 87 (WanVideoContextOptions): `context_frames` = 33
- **ประหยัดเพิ่ม**: ~1GB

### 2. ปิด LoRA ตัวที่สอง
- แก้ไข Node 354: ปิดหรือลบ `Wan21_PusaV1_LoRA`
- **ประหยัดเพิ่ม**: ~0.5GB

### 3. ลดความละเอียดเป็น 576×1024
- แก้ไข Nodes 68, 158, 270, 89, 159
- **ประหยัดเพิ่ม**: ~1.5GB

### 4. ลด Frame Buffer เหลือ 60 frames
- แก้ไข Node 75: `frame_load_cap` = 60
- **ประหยัดเพิ่ม**: ~1GB

### 5. ใช้ VAE Tile เล็กลง (192×192)
- แก้ไข Nodes 28, 497: `tile_x` = `tile_y` = 192
- **ประหยัดเพิ่ม**: ~0.5GB (แต่ช้าลง)
